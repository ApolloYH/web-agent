## 一、项目是什么

一个类 ChatGPT 的对话式 Web 应用：
- **左侧**：与后端 agent 对话（流式）
- **右侧**：预览 agent 产出物 —— Word / PDF / JSON / Markdown（只读预览，纯前端渲染）

技术栈：React 19 + Vite 7 + TypeScript + Tailwind v4；pnpm 管理；Node v24。

### 待办 / 已知限制 ⏳
- **生产部署缺反向代理**：`/noumi-api` 代理只在 `pnpm dev` 生效。生产静态产物直连仍会撞 CORS，需常驻反代（nginx / 小 node 服务）。—— 待用户决定是否补
- **SSE 文本为快照式**：Noumi 不是逐字 delta，适配器已做「快照 diff 成增量」包装，非前缀修订时会整体补发（极少见）
- 打包体积 1.5MB（含 docx-preview 的 jszip），可接受，暂不优化

## 三、关键技术结论（Noumi / langhub 后端）

| 项 | 结论 |
| --- | --- |
| Base URL | `https://www.langhub.cn/api/external/v1` |
| 鉴权 | `Authorization: Bearer nim_...` |
| 对话模型 | **项目/话题制**，非 OpenAI `/chat/completions` |
| 发消息 | `POST /projects/{pid}/topics/{tid}/messages` `{prompt}` → 返回 taskId |
| 流式 | `GET .../stream`（SSE，事件通知+快照，非逐字 delta）；`run_state.activeRun===null` = 完成 |
| 产出物位置 | **workspace 文件系统**（非消息内联），用 `tree` 发现、`files` 读取 |
| 读 JSON/MD | `files?path=X&format=text` → `content` |
| 读 Word/PDF | `files?path=X&format=binary` → `contentBase64` |
| 下载 | `GET /workspace/download?path=X` |
| CORS | 后端不返回 CORS 头，浏览器直连必被拦 → 必须走代理 |
| 鉴权兼容坑 | iframe/fetch 带不了 Bearer 头 → 适配器把产出物预取成内联，绕开 |

### API Key 权限范围（scopes）—— 当前状态（2026-07-10 起）

key 的 scope 是「保留对话/产出，禁止直接改文件」，是刻意设的安全边界，勿随意改动。

| scope | 状态 | 含义 / 影响 |
| --- | --- | --- |
| `workspace:read` | ✅ 开 | 读产出物、列文件、预览四类文档 |
| `agent:execute` | ✅ 开 | 发消息、跑 agent、出产出物（对话主链路依赖此项） |
| `projects:manage` | ✅ 开 | 新建/管理项目与话题 |
| `workspace:write` | ❌ **关** | 直接 PUT/PATCH/DELETE workspace 文件 → 返回 **403 INSUFFICIENT_SCOPE** |

含义：文件由 agent 跑任务时生成（走 `agent:execute`），**不允许绕过 agent 用 API 直接改文件系统**。
- 对 web-agent 主链路（对话 + 预览产出物）无影响。
- 附件上传（`POST .../process-files`）不属于 workspace:write，仍可用。
- 若某功能报 403 INSUFFICIENT_SCOPE，即触碰了被关闭的 workspace:write，属预期行为，不是 bug。

### 附件上传（已验证可用）

两步：① `POST /projects/{pid}/topics/{tid}/process-files`（multipart，`file=@...`）→ 返回文件 id；
② 发消息带 `attachedProcessFileIds:[id]` → agent 可读取该附件。
（也可用 `paths` 引用已有 workspace 文件；实测 agent 能准确读出附件内容。）

## 五、怎么跑

```bash
cp .env.example .env.local     # 填 NOUMI_API_KEY=nim_...（可选，也可在页面设置里填）
pnpm install
pnpm dev
```
页面 **⚙ 设置 → 选 "Noumi"** → 自动拉项目列表 → 选项目 → 发消息。话题留空自动新建。
