## 2. 产品定位与架构

这是一个类 ChatGPT 的多智能体统一 Web 入口，包含两套隔离的 Apollo：

- **新对话 / 统一入口**：独立“超级 Apollo”，负责理解工作意图、普通问答、润色、工标库检索以及 LangHub 专业业务调度。
- **助理**：独立个人 Apollo，使用固定 session、个人配置、个人技能和长期记忆，用户可以调教，不加载统一入口业务技能。
- **文件库**：统一记录两种入口产生的 Word、PDF 和图片；文件同时显示在消息中并保存到用户物理工作区。

统一入口不再使用浏览器端小模型路由。超级 Apollo 根据系统提示词和两个专用 Skill 决定直接回答或调用后端工具：

1. `query_engineering_standards`：工标库/国家标准/行业标准问答，固定 LangCore `fast` 模式。
2. `run_langhub_task`：调用 LangHub 的管控一张卡、隐患识别、施工方案审查、通知单、监理日志、监理文书、隐患分析和图纸对比等业务。

浏览器端 Mock、OpenAI 直连、Noumi 直连和 `/noumi-api` 代理已经删除，禁止按旧方案恢复。

## 3. 已完成能力

- 本地用户名密码登录、邀请码注册、显式管理员和 cookie session。
- SQLite 持久化用户、登录 session、对话、标题、消息和文件记录。
- 统一入口每个对话使用独立 Apollo Engine/session；同一用户可同时运行多个对话，侧栏显示运行状态。
- 个人助理使用用户独立配置、固定 session、记忆和技能；统一入口关闭个人记忆工具。
- `/clear` 清理当前上下文；会话自动压缩和 AI 标题总结已接入。
- 工标库真实查询、单轮检索复用和中文调用日志。
- LangHub 真实附件上传、任务轮询、成果发现、下载和结构化 `artifacts` 上报。
- 管控一张卡、隐患识别只需一张图片；施工方案审查只需一个方案文件。
- Word、PDF、JPG/JPEG/PNG/WebP 对话内预览和文件库展示；预览区域限定高度并内部滚动。
- 消息复制、附件上传、对话重命名/移动/删除、侧栏运行旋转标识。
- 文件库支持 Word、Markdown、JSON 进入统一编辑工作台；PDF 和图片保持只读。
- Markdown/JSON 停止输入 350ms 后原位保存，JSON 保存前校验格式；文件保存使用 ETag 防止覆盖外部新版本。
- Word 使用 `@agentbridges-ai/onlyoffice-browser` 纯浏览器编辑，不依赖 DocumentServer；本地 Office Host 使用独立 `127.0.0.1:5174` origin。
- 主站首屏 `load` 完成后会通过隐藏的 Office Host iframe 后台加载 X2T WASM；Service Worker 将其缓存优先保存，首次公网传输为 7.2 MB Brotli，解压后的 36.8 MB WASM 可被随后打开的 Word 编辑器直接复用。
- Apollo 注册 `document_get_context`、`document_replace_text`、`document_append_text`、`document_set_content` 工具，通过现有 SSE/审批/响应链路实时操作当前浏览器编辑器。
- 对话内文档改为 GPT 风格大卡片，提供编辑、下载、全屏入口；文档写回后卡片预览同步刷新。
- 文件库可连接本地文件夹，目录句柄保存在 IndexedDB；Word、Markdown、JSON 的打开、手动编辑和写回都在浏览器中完成，原文件二进制不上传网站。若调用云端 Agent 理解或修改当前文档，文档正文会通过编辑器工具桥返回服务器/模型。
- CLI 与 Web 展示 thinking、工具调用和必要输出，完成后自动折叠。
- CLI 代码差异采用 `highlight.js`/`cli-highlight` 和 Monokai Extended 配色；Apollo Logo、输入提示和运行星形保持品牌蓝。
- 默认权限 `ask`；`readonly` 不可被 `--yolo` 绕过；Web 的全自动模式受 `WEB_ALLOW_UNRESTRICTED` 控制。
- 正式 Node 生产服务、systemd、FRP、HTTPS 域名和持久化目录已建立。

## 6. Agent SDK 当前契约

Web 使用 `@apolloyh/apollo-agent@0.2.1`。关键宿主能力：

- `configMode: "isolated"`：只读显式配置，适合 Web 服务。
- `extraTools: ToolDefinition[]`：注册工标库和 LangHub 等宿主专用工具。
- `memoryEnabled: false`：统一入口关闭记忆工具和自动记忆注入。
- `sessionId`：每个统一入口对话恢复自己的 session。
- `onEvent`：把 thinking、工具、审批、usage、文件和错误同步到 Web。
- `cancel()`：中断当前回合并回滚尚未完成的用户消息。
- `close()`：释放 MCP 和 Engine 资源。

完整说明见 `agent/docs/sdk.md`。

## 8. 已知限制与下一步

1. **真正多用户并发尚未完成**：同一用户多对话可并行，但多用户同时跑 Agent 仍需要 per-user worker、独立进程或容器。
2. **Word Agent 目前是正文级修改**：支持读取、精确替换、末尾追加并写回 DOCX；修改后会重载浏览器编辑器。复杂样式、表格和结构化排版仍需后续 OnlyOffice 插件桥。
3. **前端主包约 1.5 MB**：主要来自 DOCX 预览依赖，当前只是构建警告，不影响运行。
4. **安全边界仍是应用层**：开放 shell、MCP 或 unrestricted 时，生产环境仍应使用容器/独立系统用户隔离。
5. **OnlyOffice 开源许可**：`@agentbridges-ai/onlyoffice-browser` 为 AGPL-3.0；主仓库已公开并加入 `AGPL-3.0-only` 许可，保留原有 ONLYOFFICE 品牌标识。
