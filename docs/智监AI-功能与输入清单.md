# 智监 AI（H5）功能与输入清单

> 逆向自前端打包文件（uni-app + Vue），记录「每个页面让用户填什么 / 交互流程」。
> 来源：https://ai-manager-h5.shippilot.shop/ ，后端 API：`https://ai-manager-api.shippilot.shop/api`（前缀 `/main/`）。
> 仅保留前端功能分析，不含机构信息和任何真实用户运行时数据。

## 概览

移动端 H5，工程监理 AI 助手。底部 4 个 tab：**首页 / 任务 / 工标库 / 我的**。
所有 AI 功能都是「提交（submit）→ 出结果（result）」两段式；隐患识别多一个「候选确认（confirm）」中间步。

登录方式：账号密码登录 + 微信登录（`/main/auth/login`、`/main/auth/login-wechat`）。所有操作绑定「当前项目」，可切换项目。

## 首页（pages/index/index）

问候语按时间变化（凌晨好/早上好/…）。展示当前项目、进行中/已完成任务数。

**AI 能力分区：**
- **三大核心智能体**：AI 工标库、AI 施工方案审查、AI 隐患识别
- **文件任务**：监理文书、AI 监理日志、AI 监理通知单、AI 监理文书生成、综合管控、管控一张卡
- **即将开放/建设中**（列在首页但未上线）：单线图对比、AI 证书识别、AI 监理文件评价、AI 语音表单、AI 旁站记录、AI 监理月报、审批流

## 1. AI 隐患识别（pages/hazard）— 两阶段

### 阶段一：提交（hazard/submit）
用户填/传：
| 输入项 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 业务项目 | 选择 | 是 | 当前项目自动带入 |
| 电网类型 | 单选：主网 / 配网 | 是 | 「识别标准依此区分」 |
| 现场图片 | 图片上传（多张） | 是（≥1 张） | 仅 jpg/png，单张有大小上限 |
| 现场说明 | 文本 | 否 | 提示：作业类型·进度阶段·特殊环境（夜间/高温/降雨等） |

流程：上传 → AI 识别候选隐患 → 用户勾选确认 → 生成正式文档。
提示：AI 识别仅辅助，最终隐患认定以人工审核为准。按钮「开始识别」。

### 阶段二：候选确认（hazard/confirm）
展示 AI 识别出的候选隐患列表，每项含：违章类别、违章代码、问题描述、依据条款、整改建议（可展开/收起、图片可放大）。
用户操作：勾选确认（至少 1 项，支持全选/全不选）→「确认并生成正式文档（N 项）」。
可「取消任务」（取消后无法继续生成）。若未识别到隐患，提示可能原因并「返回重新上传」。

### 结果（hazard/result）
隐患识别结果展示 / 生成的文档。

## 2. AI 监理通知单（pages/notice/submit）
| 输入项 | 类型 | 必填 |
| --- | --- | --- |
| 业务项目 | 选择 | 是 |
| 关联隐患识别记录 | 选择一条已确认的隐患识别记录 | 是（必选） |
| 勾选发单隐患条目 | 多选 | 是（≥1 条） |
| 项目名称 | 文本（自动带入当前项目） | — |
| 通知单编号 | 文本，如 JL-TZ-2026-007 | 是 |
| 承包单位 | 文本 | 是 |
| 事由 | 文本，如「关于临边防护不到位的问题」 | 是 |
| 检查/复查日期 | 日期 | 是 |
| 整改要求 | 文本（责任单位需执行的整改动作） | 是 |
| 整改期限 | 日期 | 是 |
| 现场图片/附图 | 图片上传（≤9 张，单张 ≤10MB） | 否 |

流程：选隐患记录 → 勾选发单条目 → 填通知单信息 → AI 生成监理通知单。按钮「生成监理通知单」。

## 4. AI 监理文书生成（pages/supervision-doc/submit）
先选文书类型：**监理工作总结**（阶段/竣工）/ **监理规划** / **监理实施细则**。

| 分组 | 输入项 |
| --- | --- |
| 签批信息（8 项） | 部门、编制人、编制日期、审核人、审核日期、批准人、批准日期、状态（初版/修订中/待审核/已批准） |
| 工程信息（2 项） | 周期（如 2026-01-01 至 2026-12-31）、工程简介 |
| 施工概况 | 文本，当前主要施工内容/进度/现场重点关注事项（必填） |
| 补充附件 | 文件上传（≤20MB，组织架构图/施工记录/问题台账等，可选） |

按钮「提交生成文书」。

## 8. AI 工标库问答（pages/standard/chat + history）
聊天式交互，输入框「输入您的工标问题...」。接入南方电网企标/电力行标/档案管理标准。
支持：新建会话、历史会话、复制回复、重试、反馈。
二期功能（未开放）：上传工标文件做格式处理、上传标准导入企业规范库。
提示：暂仅支持企业标准查询/使用。

## 其他页面
- **任务**（task/list、task/detail）：任务列表与详情，各 AI 任务的进度/结果入口。
- **我的**（mine）：个人信息（profile）、修改密码（change-password）、切换项目（project/switch）、消息中心（notification/list）。
- **通用**：文章页（common/article）。

## 输入类型总结
- **图片上传**：隐患识别、通知单附图、日志现场图、管控卡现场照片（jpg/png/webp）
- **文档上传**：方案审查（doc/docx/pdf）、单线图（图片/pdf）、文书附件、管控卡辅助资料
- **表单文本**：通知单、监理日志、文书签批信息字段最多
- **AI 辅助输入**：监理日志「今日主要施工内容」支持语音口述 + AI 润色
- **跨模块引用**：通知单/日志可引用「已确认的隐患识别记录」，自动带入隐患明细
