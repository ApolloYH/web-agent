# 智监业务关键输入

只校验会实质影响结果的字段；用户已在对话或附件中提供的内容不得重复询问。

- `risk_card`：一张现场图片即满足全部启动条件，只传这一张图片并立即执行。项目/作业名称、地点、施工内容、日期和风险信息仅在用户主动提供时带入，不得追问。
- `supervision_notice`：项目名称、通知单编号、承包单位、事由、检查日期、整改要求、整改期限；相关隐患和图片。
- `supervision_log`：项目名称和编号、日期、天气/气温、施工内容及部位、人员机械、质量安全情况、问题及处理。
- `supervision_document`：文书类型、项目名称、编制目的或范围、已有项目资料/模板；具体章节要求如有。
- `hazard_analysis`：一张现场图片即满足全部启动条件，只传这一张图片并立即执行，不得追问主网/配网或现场说明。识别结论必须提示人工确认。
- `plan_review`：一个需要审查的施工方案文件即满足全部启动条件，只传这个方案并立即执行。项目/方案名称和审查重点仅在用户主动提供时带入，不得追问；没有方案文件时不得执行正式审查。
- `drawing_compare`：改造前和改造后两套图纸，并能区分版本；缺任一版本时不得执行。

附件路径必须来自当前用户消息中列出的工作区路径，例如 `uploads/文件名.docx`。
