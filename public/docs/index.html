Apollo · 个人超级助手

把想法、文件和网页交给 Apollo，让工作真正完成。

Apollo 能持续对话、处理文件、编辑文档、操作浏览器并创建可发布的网站。你只需要说明目标，过程和结果会在同一个工作区里完成。

立即使用阅读快速开始

快速开始

从一个清晰目标开始，不需要学习特殊提示词。

  1. 登录 Apollo
    使用管理员提供的账号，或在开放注册时使用邀请码创建账号。
  2. 创建新对话
    直接描述要完成的工作，也可以上传图片、文档或数据文件。
  3. 确认必要操作
    涉及写文件、浏览器点击等操作时，根据当前审批模式确认后继续。
  4. 查看并继续修改
    产出的文件、网站和浏览器过程会留在工作区，可继续对话迭代。
推荐描述方式：说明“目标、已有资料、希望的输出”。例如：“整理这份会议记录，生成一份带负责人和截止日期的行动清单。”

对话与任务

新对话

每个对话拥有独立上下文，适合处理一个明确主题或项目。

助理

用于长期协作。Apollo 会在允许的范围内使用个人配置、技能和记忆。

附件

可随消息上传图片、PDF、Word、表格、文本、Markdown 和 JSON。

持续执行

离开当前对话不会自动中断任务，侧栏会显示仍在运行的对话。

常用操作

操作用途
停止中断当前正在生成或执行的回合。
/clear清理当前对话上下文，保留其他对话。
重命名 / 归档 / 删除在侧栏的对话菜单中管理历史记录。

文件库与文档工作台

“文件库”集中展示 Apollo 生成的文件，也可以连接浏览器授权的本地文件夹。

Word

直接在网页内预览、编辑、问答并保存；复杂排版仍可手动调整。

Markdown / JSON

支持阅读、编辑、保存和下载，Markdown 可渲染常见安全 HTML。

PDF

支持预览、提取文本、搜索内容和向 Apollo 提问。

图片

可在工作区预览，并作为对话附件交给任务使用。

本地目录句柄保存在当前浏览器。手动查看和编辑时，原文件不会自动上传；只有你要求云端 Apollo 理解或修改内容时，相关正文或清单才会参与任务。

RAG 知识库

打开侧栏“RAG”，创建知识库并选择适合资料结构的处理模板。上传完成后，在普通对话或助理中说“根据我的知识库回答”，Apollo 会检索原文并标注文档来源。

通用与问答

通用文档按语义边界切分；问答对会把问题和答案保留在同一分段。

手册与书籍

按标题、编号、章、节和卷组织长文，减少上下文断裂。

论文与法规

识别论文结构，或按“第几条”保留完整法规条款。

表格与演示

表格按行、演示文稿按页面处理；短资料也可选择整篇文档模板。

  1. 新建知识库，填写名称并选择一种文档处理模板。
  2. 上传 PDF、Office、图片、文本、Markdown、CSV、HTML 或 JSON 文件。
  3. 使用“召回测试”检查原文命中情况，再回到对话中提问。
检索使用关键词与语义向量混合召回，并进行相关性重排。普通文本优先在 Apollo 本地解析；扫描件、旧版 Office、演示文稿和图片等本地无法读取的文件会发送到已配置的 MinerU 服务解析。

创建并发布站点

打开侧栏“站点”，先进入已有站点预览页。点击“创建站点”后，就能在左侧持续对话、右侧实时查看结果。

  1. 发送参考网页、资料或直接描述用途、内容和风格。
  2. Apollo 创建第一版后，右侧会出现可点击的实时预览。
  3. 继续描述修改，确认后重新部署。
  4. 发布成功后获得可直接分享的公开 URL。

浏览器操作

Apollo 有两条隔离的浏览器链路。默认使用托管浏览器；只有任务意图明确需要你的浏览器、登录状态或当前标签页时，才使用用户 Chrome。

托管浏览器

无个人 Cookie 的隔离环境，适合公开网页搜索、资料整理和常规网页任务。过程会显示在右侧实时画面。

用户 Chrome

通过 Apollo Browser Bridge 操作你明确选择的标签页，可复用当前登录状态。接管期间页面会显示鼠标和发光边缘。

使用自己的浏览器

  1. 安装管理员提供的 Apollo Browser Bridge 扩展。
  2. 打开要操作的网页，并点击扩展图标连接当前标签页。
  3. 在任务中明确表达使用自己的浏览器，例如“用我的浏览器整理当前页面”。后续连续指令会结合当前任务意图判断。
  4. 任务完成或停止后，Apollo 会结束接管提示。

在 IM 中使用 Apollo

助理支持飞书、企业微信、钉钉、微信和 Telegram。连接后，IM 消息会进入同一个 Apollo 助理，继续使用已有记忆、工作区和托管浏览器。

飞书

创建企业自建应用并启用机器人与消息事件，填写 App ID、App Secret 和允许使用者的 Open ID。

企业微信

创建智能机器人，填写 Bot ID、Secret 和允许使用者的企业微信 user ID。

钉钉

创建企业内部应用，为机器人选择 Stream 模式，填写 Client ID、Client Secret 和 Staff ID。

微信

点击“获取二维码”，用手机微信扫码确认。Apollo 会创建独立的微信机器人身份，当前以单聊为主。

Telegram

通过 @BotFather 创建机器人，填写 Bot Token 和允许使用者的数字 ID。

  1. 打开 Apollo 助理右上角“用户中心”，进入“IM 接入”。
  2. 选择渠道，填写平台凭据和允许使用的用户 ID；微信直接扫码。
  3. 打开渠道开关并保存，状态变为“已连接”后即可发送文字消息。
安全提示：所有渠道默认拒绝白名单之外的用户；Secret、Token 保存后不会返回浏览器。IM 中无法操作你当前的 Chrome 标签页，但可以使用 Apollo 托管浏览器。飞书、企业微信和钉钉使用官方长连接，不需要配置公网 Webhook。

权限与隐私

账号隔离

不同用户不共享配置、对话、记忆、会话路径和文件工作区。

审批模式

审批模式会在关键操作前询问；全自动模式仅在管理员允许时可用。

浏览器边界

扩展只操作已连接的 HTTP/HTTPS 标签页，托管浏览器不共享你的 Chrome Cookie。

系统边界

网页端永久拒绝系统命令、工作区越界、敏感文件读取和未授权的扩展能力。

常见问题

为什么 Apollo 没有直接操作我的浏览器？

默认会使用隔离的托管浏览器。需要登录状态或当前页面时，请确保扩展已连接，并在任务中表达使用自己浏览器的意图。

关闭页面后任务会停止吗？

切换页面或对话通常不会停止运行；主动点击“停止”或服务重启才会中断当前回合。

生成的网站在哪里？

发布后会显示在“我的站点”中。点击站点卡片可以继续预览和修改，也可以在新窗口打开公开链接。

文档修改会覆盖原文件吗？

保存操作会写回当前文件。重要文件建议保留备份；遇到不兼容的复杂 Word 内容时，系统会降级为只读，避免破坏原文件。
