01 / OVERVIEW

一个工作区，连接任务的完整上下文

Apollo 不只是回答问题。它会围绕同一个目标调用文件、知识、网页和工具，并把执行过程与最终产物留在当前对话中，方便继续修改。

持续任务

在对话里拆解和执行复杂任务，切换页面后任务仍可继续运行。

文件工作台

上传、预览、编辑和生成文档，让 AI 直接围绕真实文件工作。

知识与图谱

双路检索文段、实体与关系路径，让回答兼顾原文证据和知识脉络。

浏览器执行

使用隔离浏览器，或在授权后操作你明确连接的 Chrome 标签页。

站点制作

一边对话一边预览网站，可选中多个页面元素并准确引用修改。

多端协作

从网页或常用 IM 发起任务，共享同一个助理、记忆和工作空间。

02 / GETTING STARTED

三步开始你的第一个任务

直接描述目标即可。明确资料范围、期望产物和限制条件，通常能得到更稳定的结果。

  1. 创建对话登录后点击“新对话”，用一句话描述要完成的事情，而不只是询问一个知识点。
  2. 补充上下文上传文件、选择知识库，或说明要访问的网页。敏感操作会按当前审批模式请求确认。
  3. 检查并继续在活动面板查看执行过程，在消息中预览产物；不满意时直接基于当前结果继续修改。
推荐写法：“读取这些制度文件，按部门整理关键要求，标注原文出处，最后生成 Word 报告。”目标、范围、证据和产物都明确，比“帮我看看文件”更容易一次完成。
03 / CHAT & TASKS

对话负责目标，活动面板负责过程

主对话保持干净，只展示任务交流与可用结果；思考、工具调用和执行状态集中在活动面板中。需要细看时展开，不需要时不会打断阅读。

围绕结果持续迭代

  • 任务不中断：切换到文件、知识库或其他对话，当前回合通常仍在后台执行。
  • 状态更准确：单个工具失败不会把已完成的整体任务误标成“执行异常”；最终状态以任务结果为准。
  • 过程可追踪：活动面板区分正在思考、工具执行与已完成记录，可查看每一步详情。
  • 阅读不被打断：页面只在你停留在最新消息附近时自动跟随；向上阅读后不会被强制拉回底部。
  • 产物可识别：报告、数据文件和其他结果带有名称与类型，便于预览、下载和后续引用。

统一管理常用资料

  • 常用格式：支持 Word、PDF、Markdown、TXT、JSON、代码和图片等内容的预览或处理。
  • 网站与本地目录：站点文件和授权的本地文件夹都能进入工作区，不必反复上传。
  • 所见即所得：可以在分屏中打开文档，保存会写回当前工作区文件；复杂格式不兼容时自动只读。
  • 大文件保护：超大文本仅载入前 512 KB 预览，并进入只读模式，避免浏览器卡死或把截断内容误覆盖回原文件。
重要资料建议保留备份。如果 Apollo 提示当前内容为只读，请让它另存为新文件，而不是强行覆盖原文件。
05 / RAG KNOWLEDGE

既检索原文，也理解知识之间的关系

Apollo 的知识库组合 WeKnora 文段检索与 LightRAG 知识图谱。前者找到最接近问题的证据段落，后者补充实体、本体、关系和检索路径，再由 Apollo 结合两路结果作答。

从文件到可追溯答案

  • 双路召回：同时返回匹配段落、相似度得分、图谱实体、关系和路径，不只给出一段模糊摘要。
  • 处理进度：文档列表展示解析、切片和图谱抽取阶段，便于判断任务是在运行、排队还是失败。
  • 图谱探索：彩色节点区分实体类型，点击节点查看介绍和关联关系，画布支持滚动、缩放与拖动。
  • 性能边界：大图谱采用逐帧布局并限制单次可视节点，加载时页面仍可操作；需要时再继续缩小范围。
  • 用量可见：知识库设置显示 Token 统计，并可在高级配置中调整允许的抽取并发。

建立并使用知识库

  1. 创建知识库在“知识库”中新建集合，填写名称和说明，帮助 Apollo 判断资料用途。
  2. 上传并等待处理查看每份文档的两路状态与切片进度；失败的引擎可单独重试，不必重复上传。
  3. 检索或加入对话先用召回测试检查匹配段落和得分，再让 Apollo 基于指定知识库完成回答或报告。

从想法到可访问站点

  • 实时预览：生成过程和页面结果并排展示，点击预览区域即可操作真实页面。
  • 多元素选择：一次选择标题、按钮、卡片等多个元素，它们会作为结构化引用进入输入框。
  • 结构理解：Apollo 获得元素路径和页面上下文，修改更精准，不依赖你猜测组件名称。
  • 持续迭代：发布后站点保留在“我的站点”，可再次打开、继续对话修改并重新发布。
操作技巧：先选中需要修改的元素，再描述“改什么、不要动什么”。多个相关元素可以一起选择，不必逐个发起对话。

使用自己的浏览器

  1. 安装扩展安装管理员提供的 Apollo Browser Bridge，并保持 Apollo 网页端已登录。
  2. 连接标签页打开目标网页，点击扩展图标，只授权当前需要操作的标签页。
  3. 明确下达任务例如“用我的浏览器整理当前页面”。页面上的接管提示会展示当前状态。

10 / SAFETY

能力越完整，边界越需要明确

Apollo 将用户数据、执行权限和浏览器环境分层隔离。审批模式负责在人机之间确认关键操作，系统边界负责拒绝网页端不应拥有的能力。

01

多用户隔离

不同账号的对话、配置、记忆、文件工作区和运行会话彼此隔离；停用账号会撤销现有会话。

02

操作审批

审批模式会在关键操作前请求确认。全自动模式只在管理员允许的范围内开放。

03

浏览器边界

托管浏览器不共享你的 Chrome Cookie；扩展只操作你明确连接的网页标签页。

04

工作区边界

网页端拒绝系统命令、敏感文件读取、工作区越界和未授权扩展能力，避免任务突破可控范围。

12 / FAQ

常见问题

关闭或切换页面后，正在执行的任务会停止吗？

通常不会。切换页面、切换对话或暂时关闭浏览器不会主动停止当前回合；点击“停止”、服务重启或会话被管理员撤销时才会中断。回来后可在原对话继续查看结果。

为什么 Apollo 没有直接操作我的 Chrome？

默认会使用隔离的托管浏览器。需要复用登录状态或当前页面时，请先通过 Browser Bridge 连接目标标签页，并在任务中明确说明使用自己的浏览器。

知识库为什么同时显示 WeKnora 和 LightRAG？

两者负责不同检索能力：WeKnora 提供切片和文段召回，LightRAG 抽取实体关系并提供图谱上下文。某一路暂时失败时可单独重试；回答会清楚展示实际使用的引擎状态。

召回测试里的相似度得分是什么意思？

得分用于比较当前查询与候选段落的匹配程度，通常越高越相关，但不等同于事实正确率。建议同时检查段落内容、来源以及图谱关系路径。

为什么大型知识图谱不会一次显示全部节点？

为了保持页面可以滚动、点击和缩放，图谱会限制单次可视节点并分帧计算布局。先通过搜索或检索结果缩小范围，比一次渲染全部关系更容易找到有用信息。

打开很大的 TXT 或 JSON 文件时为什么是只读？

为避免浏览器长时间卡顿，大文本只加载前 512 KB 用于预览，并自动进入只读模式。这样也能避免把截断后的内容误保存回原文件。需要完整处理时，可以让 Apollo 在后台读取并另存结果。

文档修改会覆盖原文件吗？

可编辑模式下，保存会写回当前工作区文件。重要资料应保留备份；复杂 Word 内容或大文件可能自动降级为只读，此时建议生成新的结果文件。

生成的网站在哪里？

发布后的站点会出现在“我的站点”。点击卡片可以继续预览、选择页面元素并修改，也可以在新窗口打开公开地址。

注册码和邀请码有什么用途？

开放注册时，新用户需要有效注册码完成账号创建。管理员可以在后台生成、停用并查看注册码使用状态，普通用户看不到这部分运营功能。

IM 渠道能使用我电脑上的浏览器吗？

不能。IM 任务可以使用 Apollo 托管浏览器，但不会远程接管个人 Chrome。用户 Chrome 必须在网页端通过扩展主动连接具体标签页。
